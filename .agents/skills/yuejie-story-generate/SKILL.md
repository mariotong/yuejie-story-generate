# 岳姐故事版生成

## 目标

生成可直接进入制作的岳姐短视频故事版：标题、总时长、9:16 竖屏格式、6-10 个镜头、每个镜头的情绪推进、台词/旁白、转场逻辑，以及每个镜头首帧/尾帧的 Codex image2 图片生成提示词。默认视觉方向是明亮、清楚、温暖、适合中老年手机观看。

这个 skill 只负责故事版和分镜生成，不负责伪造真人记录。用户如果要写岳姐公众号文章、口吻校准或人物表达风格，可以结合 `yuedongqing-writing-style`；本 skill 仍然专注视频结构和画面生成。

## 参考文件导航

- `references/storyboard-format.md`：故事版表格字段、镜头弧线、时长限制、岳姐人设出镜规则。生成或重写完整故事版时读取。
- `references/article-to-storyboard.md`：把长文章转成故事版的方法。用户给文章、公众号文案、长段观点，希望规划/生成故事版图片时读取。
- `references/visual-image2.md`：明亮中老年视觉方向、Codex image2 提示词公式、首帧/尾帧区别、场景细节。写图片提示词或生成故事版图片/单图素材时读取。
- `references/shot-control-prompts.md`：Grok/图生视频用的分镜控制提示词格式。用户要求“给分镜控制提示词”“结合故事版第N个镜头给分镜控制提示词”“镜头运动/动作设计/秒数口播”时读取。
- `references/image-to-6s-oral-prompt-lessons.md`：单张人物参考图转 6 秒口播分镜提示词的经验。用户发图问“这张适合啥”“直接给提示词”“家庭关系口播”“分镜口播提示词”时读取；若用户说“直接给我提示词”，只输出可复制提示词代码块，不要长篇分析。
- `references/yuejie-one-minute-oral-topic-validation.md`：岳姐一分钟口播的已验证爆款母题与选题优先级（情绪养生、父母退出、过度负责、全家说明书、夫妻心疼等）。用户问“还可以讲啥话题”“有人爆过吗”“一分钟口播选题”时读取。
- `references/yuejie-family-oral-topic-patterns.md`：岳姐家庭关系口播选题、6秒总览、60秒固定机位结构、爆点句库和用户“直接给提示词/口播/总览”的交付偏好。用户问家庭关系/中老年爱看什么/6s分镜/一分钟口播时优先读取。
- `references/grok-short-video-split-patterns.md`：Grok 额度或时长不足时，把 8-10 秒镜头拆成 5 秒 A/B 子镜头的确认范式。用户说“1A/1B/6A/6B/分镜头1/分镜头2”时读取。
- `references/review-checklist.md`：生成后自审规则、45+ 女性完播/转发审查、修改意见格式。生成完整故事版后必须读取并执行。
- `references/single-image-batches.md`：故事版确认后生成单独图片的批次规则，以及 `storys/` 归档规则。用户要求“前五个镜头首尾两帧”“十张图”“只要内容不要文字”或生成完图片要保存数据时读取。
- `references/agent-reproduction-prompt.md`：给其他 agent 使用的故事版图片复现提示词模板。用户要求“给我一个生成故事版的提示词给其他agent使用”“让别的agent复现这个图”时读取。

只读取当前任务需要的参考文件。不要为了生成单独图片重新加载完整故事版规则，除非需要回查镜头内容。

## 工作流程

1. 判断用户目标：选题/标题、发布平台、目标观众、情绪方向，以及用户要完整表格、只要图片提示词、修改已有故事版，还是生成单独素材图。
2. 用户给长文章或公众号文案时，先读取 `references/article-to-storyboard.md`，把文章压成可视化故事弧线，再读取 `references/storyboard-format.md` 和 `references/visual-image2.md` 生成完整故事版。
3. 围绕一个清晰的情绪转折搭故事：日常负担 -> 被忽视或习惯性依赖 -> 看见代价 -> 建立边界 -> 温暖释放。
4. 默认写 6-10 个镜头。每个镜头通常 6-10 秒，且单个镜头绝对不能超过 10 秒；这是 Grok 生成限制。
5. 完整故事版里至少要有一个镜头明确出现岳姐本人；其他镜头可以通过她的手部、背影、生活痕迹或家人反应来服务人设。
6. 每个镜头同时给出故事内容和生成内容：画面内容、首帧/尾帧 image2 提示词、时长、台词/旁白、转场、场景细节、生成策略。
7. 生成完整故事版后，读取 `references/review-checklist.md` 并追加自审。自审必须检查合理性、对白自然度、转场、镜头数量、Grok 时长限制、岳姐出镜、image2 可行性、45+ 女性完播/转发潜力和明亮清楚。
8. 用户要求给某个镜头的分镜控制提示词时，读取 `references/shot-control-prompts.md`，按“基础画面 + 镜头运动 + 动作设计和秒数口播 + 画面情绪”输出，并主动审查每段口播是否能在对应秒数里自然说完。
8a. 用户要求 1A/1B、6A/6B、“分镜头1/2”或说明 Grok 额度/时长不足时，同时读取 `references/grok-short-video-split-patterns.md`，把原 8-10 秒镜头拆成 5 秒 A/B 子镜头；每个子镜头只承载一个动作变化和一个情绪落点，仍必须写明 `@image1/@image2`、旁白/对白秒点、说话人控制和合理性审查。
9. 用户确认故事版图片没问题后，若要求生成单独图，读取 `references/single-image-batches.md`，按批次调用 Codex image2。Codex image2 一次最多 10 张。
10. 每次生成图片后，都要在 `/home/ubuntu/.hermes/skills/yuejie-story-generate/storys` 下为本次故事新建一个目录，归档图片、提示词、故事版文本和图片清单数据。
11. 用户要“给其他 agent 的生成提示词”时，读取 `references/agent-reproduction-prompt.md`，输出一段可复制的完整提示词，而不是直接生成图片。

## 默认输出格式

```text
标题：{强情绪标题}
总时长：约{秒数}秒
镜头数：{6-10}个
画面比例：9:16竖屏
风格：真实/生活化/中老年爱看/明亮清楚/温暖但不悬浮

| 镜头 | 画面内容 | 首帧 image2 提示词 | 尾帧 image2 提示词 | 时长 | 时间轴：台词/旁白 | 转场/情绪承接 | 其他：场景/细节 | image2 生成策略 |
| ... |

结尾字幕：{一句落点}
背景音乐：{温度/节奏}
生成注意：{人物一致性、服装、光线、镜头语言}

自审：
{按 review-checklist.md 输出}
```

## 迭代触发

- 用户说“不够打动人、太假、太文学”：强化日常触发点、具体动作和道具，减少说教。
- 用户给一整篇文章并问“怎么生成故事版/怎么规划故事版图片”：先做文章拆解，再给镜头规划；不要逐段复述文章。
- 用户说“转场不自然、镜头数量有问题”：重排镜头承接，必要时增删镜头。
- 用户说“中老年爱看、明亮一点”：提高画面亮度、脸部清晰度、构图可读性和色彩亲切感。
- 用户说“一个镜头不能超过十秒、Grok限制”：直接重写为单镜头不超过 10 秒的新故事版。
- 用户说“打造岳姐人设、岳姐出现不能少、可以增加镜头”：确保至少一个镜头出现岳姐本人，并强化她的动作、表情或生活细节。
- 用户说“给分镜控制提示词、结合故事版第N个镜头给分镜控制提示词、结合第一个/第二个/第三个镜头给提示词、Grok/图生视频用”：先定位故事版对应镜头，再输出单镜头分镜控制提示词，不要只给静态图片提示词。
- 用户已经确认一种提示词格式后，后续说“镜头1/镜头2/按照之前写的/用 image1”时，必须沿用前面格式：只输出可复制的提示词代码块，包含 `@imageN`、基础画面、人物一致性/禁止项、`分镜控制`、按秒点拆的字幕；不要重新解释方法、不要扩写成60秒、不要额外分析。若用户明确要求“每个镜头10s，总共六个镜头”，单镜头提示词固定为10秒，按 0-3 / 3-7 / 7-10 秒承载一句主口播与两句补充字幕。
- 用户说“前五个镜头首尾两帧、十张图、只要内容不要文字”：直接进入单独图片批量生成，不要重新输出完整故事版。
- 用户说“保存图片数据、放到storys、归档这次story”：在 `storys/{故事目录}` 中保存本次图片和元数据；如果已经生成了图片，也要补建目录并整理进去。
- 用户说“给我一个生成故事版的提示词给其他agent使用、让别的agent复现这个图、复现故事版图片的prompt”：输出 agent 复现提示词模板，不调用图片生成。
- 用户在已给文章/已确认提示词后说“生成故事板图片、生成故事版图片、直接出图”：不要再解释规划或只返回 prompt，直接调用 `image_generate`，使用 9:16/portrait，prompt 里包含完整标题、元信息、9 行镜头内容、底部金句和禁止项；生成后回复必须包含 `MEDIA:/absolute/path`。

## 生成完整故事版表图的注意事项

- 一张图承载 9 行表格时，中文小字可读性是主要风险。prompt 要强调“完整可读的故事版表、表格中文尽量清晰、不是九宫格海报”。
- 若用户要“故事板图片/故事版表图”，优先生成一张完整竖版表图；若用户后续反馈看不清或要视频素材，再拆成“每个镜头首帧+尾帧”的单独图片批次。
- 不要用真实医疗诊断单、可读电话号码、聊天记录等细节；体检单、手机屏幕可以出现，但文字应模糊不可读。

## 安全边界

不要声称生成的图片或脚本是真实岳东青记录、真实家庭事件或医疗/法律建议。不要伪造私人证据、聊天截图、单据、诊断结果，也不要对可识别真人做诽谤性描写。
