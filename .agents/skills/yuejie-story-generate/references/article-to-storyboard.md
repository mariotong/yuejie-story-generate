# 文章转故事版

当用户给一整篇文章、公众号文案或长段观点，要求生成故事版、规划故事版图片、转分镜表时使用本文件。

## 核心原则

文章不是逐段搬进镜头。先把文章压成一个可视化短片：

```text
人物：谁在承受问题
冲突：她为什么撑不住
代价：身体/情绪/家庭关系出了什么问题
转折：哪句话或哪件事让她醒过来
行动：她做了一个什么边界动作
结果：家人怎么变化，她自己怎么松一口气
金句：最后落在哪一句可转发的话
```

## 拆解步骤

1. **提炼标题观点**：从文章里找最适合短视频的反转句。标题要有冲突和观点，例如“别再说‘我来吧’，你不是贤惠，是把自己放到最低成本”。
2. **确定主角**：文章里有具体人物就保留，例如“林姐”；但故事版画面仍要服务岳姐账号，至少一个镜头出现岳姐本人。
3. **选 6-10 个可视化场景**：只选能拍出来的动作和物件，少拍抽象道理。
4. **把观点变动作**：把“她把自己耗空了”转成“病床上挂点滴，手机还在响”；把“家人默认她会处理”转成“厨房里一边炒菜一边被三个人叫”。
5. **安排情绪弧线**：日常负担 -> 医院代价 -> 岳姐点醒 -> 回家试着不做 -> 家人乱一阵 -> 家开始重新分担 -> 她释放。
6. **控制镜头时长**：每镜头 6-10 秒，绝不超过 10 秒。
7. **少讲道理，多给场面**：旁白只点关键句，不要把文章整段塞进台词。
8. **结尾保留可转发金句**：最后一句要像观众愿意转给姐妹的话。

## 文章信息筛选

保留：

- 具体人物：林姐、岳姐、丈夫、孩子、老人。
- 具体物件：体检单、病床、点滴、手机、校服、燃气卡、药、厨房、饭桌。
- 具体动作：拿体检单、掉眼泪、接电话、挂点滴、孩子煮面、丈夫煎蛋、不做饭。
- 关键台词：“我来吧”“我快撑不住了”“那你倒下了，他们就会了？”“我不动，这个家也不会塌。”

压缩或改写：

- 大段观点输出。
- 连续排比句。
- 抽象说教。
- 重复表达“她很累”的段落。

## 推荐镜头结构

对这类“女人别总说我来吧”的文章，推荐 9 镜头、约 72-78 秒：

1. 林姐拿体检单来找岳姐，眼泪掉下来。
2. 厨房/家庭日常，她一边做饭一边被家人连续叫。
3. 家庭事务蒙太奇：校服、缴费、药、燃气卡都找她。
4. 半夜胃疼进医院，医生问长期饮食睡眠压力。
5. 病床上挂点滴，手机仍不断响，家人第一反应还是问她怎么办。
6. 岳姐坐在床边或茶桌前点醒她：“你不是生病才可怜，是早把自己活成了工具。”
7. 她回家后第一次说：“今天我不做饭，你们自己安排。”
8. 家里乱一阵：孩子煮面、丈夫煎蛋、厨房乱，但大家开始动手。
9. 第三周，家人开始分担；她在阳台喝水，终于松一口气。

## 这个文章的故事版定位

标题方向：

```text
女人别再说“我来吧”，这不是贤惠，是把自己放到全家最低成本
```

总时长：

```text
约72-78秒，9个镜头，每个8-9秒，不超过10秒。
```

视觉风格：

```text
9:16竖屏，真实生活化，中老年爱看，明亮清楚，医院和家庭场景都要脸部可见，不要灰暗苦情。
```

核心情绪：

```text
不是控诉家人坏，而是拍出“所有人都习惯她兜底”，以及她终于把责任还回去后的松动。
```

结尾字幕方向：

```text
别再轻易说“我来吧”。说一次是帮忙，说十年就是消耗。
```

## 输出规划时的格式

用户问“怎么规划生成故事版图片”时，先输出规划，不必立刻生成图片：

```text
我会这样拆：
1. 核心冲突：...
2. 主角关系：...
3. 视觉主线：...
4. 镜头规划：...
5. 生成故事版图片前的检查：...
```

如果用户接着说“生成故事版”，再按完整故事版表格输出。
