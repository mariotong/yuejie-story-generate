# 分镜控制提示词

这个文件用于生成 Grok / 图生视频 / 视频镜头控制提示词。它和静态 image2 提示词不同：不仅描述画面，还要写镜头运动、动作设计、秒数口播和画面情绪。

## 目录

- 输出结构
- @image1/@image2 首尾图引用规则
- 短时长拆分规则
- 旁白主导规则
- 岳姐专门口播总结规则
- 说话人控制规则
- 基础画面提示词写法
- 镜头运动写法
- 动作设计和秒数口播
- 口播时长审查
- 画面情绪写法
- 满意模板

用户常见口令：

```text
结合故事版第一个镜头给分镜控制提示词。
结合故事版第三个镜头给分镜控制提示词。
结合故事版第N个镜头给分镜控制提示词。
给镜头2的分镜控制提示词。
第二个镜头分镜头1。
第三个镜头拆成两个镜头，给第一个分镜头。
这个镜头拿去给 Grok / 图生视频用。
```

当用户说“结合故事版第几个镜头”时，先从当前对话或已确认故事版里定位对应镜头，再按本文件结构生成。不要重新写完整故事版，也不要输出其他镜头。

如果前一个任务（例如批量生图）被用户新消息打断，先用一句话交代已完成/未完成状态，然后立刻回答用户当前要的分镜控制提示词；不要继续执行旧任务抢占当前需求。

## 李哥/磊哥快捷输出偏好

当用户说“直接给我提示词”“直接给分镜口播提示词”“不要解释，给提示词”时，**只输出一个可复制的代码块提示词**，不要先解释“适合什么”、不要给多个备选、不要写方法论。若必须判断方向，把判断融入提示词里的画面情绪/口播台词，不在代码块外展开。

当用户先问“这个适合啥”时，可以先用 1-2 句给方向；但如果同一句里又说“直接给提示词/分镜口播提示词”，仍以可复制提示词为主，外部说明最多一句。

当李哥/磊哥连续索要“镜头1 / 镜头2 / 镜头3”或明确说“只给我镜头X提示词”时，只输出对应单镜头提示词代码块，不要重复总方案、不要补其他镜头、不要解释。若用户说“按照之前写的”，保持上一版提示词结构、字段和语气，只替换时长、镜头编号、@image 引用与对应口播。

当用户提供爆款原文并纠正“口播稿跟我提供要一致”时，必须严格沿用用户提供的文稿，不要擅自改成岳姐版、人设版、去AI版或更温柔的改写版。镜头提示词里的“字幕”和“口播台词”都应来自原文顺序；若用户修改为“25s三个镜头”或“四个镜头每个6s”，只做最小切分/删减以适配时长，不重写观点和措辞。

## 输出结构

按这个结构输出。用于 Grok / 图生视频时，必须把首尾参考图写成 `@image1` 和 `@image2`，并在人物、首尾图、镜头运动、逐秒画面里反复绑定这两个引用，避免模型换人、换场景或搞错谁在说话：

```text
镜头{编号}｜{时长}秒｜{场景/动作标题}

参考图控制：
@image1 是本镜头首帧图：{首帧画面，一句话写清人物、动作、场景、关键道具}。
@image2 是本镜头尾帧图：{尾帧画面，一句话写清人物、动作、场景、情绪落点}。
视频必须从 @image1 自然运动到 @image2，不要换人、不要换场景、不要换衣服。

视频规格：
{9:16、真实生活化、光线、禁止文字字幕水印、禁止可读信息等}

人物一致性：
@image1 里的{人物名}：{年龄感、发型、服装、状态、所在场景}。
@image2 里的{人物名}：必须和 @image1 是同一个人，同样的脸型、年龄感、发型、服装和场景，不要变年轻，不要网红脸，不要换人。

画面人物控制 / 说话人控制：
{说明画面里出现哪些人；哪些人只做画外音；谁开口时谁闭嘴；旁白出现时画面人物必须闭嘴，不要对口型。}

首尾图关系：
@image1 首帧：{更具体的起点画面}。
@image2 尾帧：{更具体的终点画面}。
视频必须从 @image1 的状态开始，逐渐发展到 @image2 的状态。

镜头运动：
{镜头怎么从 @image1 的构图开始，怎么推进/跟随/停顿，最后怎么落到 @image2 的状态。}

逐秒画面 + 声音控制：

0.0s-1.0s
画面：
从 @image1 开始。{动作、表情、道具、谁闭嘴}

声音：
{环境声/不要人声/具体声音}

1.0s-2.0s

{动作变化；如果有人说话，写清谁嘴唇动、谁闭嘴、谁不入镜}

口播精确出现：
{start}s-{end}s
{说话人}说：
“{短台词}”

说话人控制：
{例如：这句必须是林姐说；画面里只有林姐嘴唇动；其他人不要入镜；旁白时林姐闭嘴。}

...

画面情绪：
{这个镜头真正要拍出的情绪，不要只写剧情。}

最终要求：
视频必须从 @image1 的首帧自然运动到 @image2 的尾帧。@image1 和 @image2 中的核心人物必须保持同一人。不要新增无关人物入镜。不要换场景。不要换衣服。不要出现任何文字、字幕、水印、可读屏幕或单据。
```

## @image1/@image2 首尾图引用规则

当用户提供首帧图和尾帧图，或要求“首帧/尾帧图生视频”“Grok 图生视频”“把首图尾图写进提示词”时，必须使用以下写法：

- 用 `@image1` 表示首帧图，第一段就写明：`@image1 是本镜头首帧图：...`。
- 用 `@image2` 表示尾帧图，第一段就写明：`@image2 是本镜头尾帧图：...`。
- 人物一致性段落里必须写：`@image1 里的{人物名}` 和 `@image2 里的{人物名}`，并声明两者必须是同一个人。
- 镜头运动必须写“从 @image1 的构图开始”以及“最后自然落到 @image2 的状态”。
- 逐秒画面开头必须写“从 @image1 开始”；结尾必须写“画面自然接近 @image2”。
- 最终要求必须再次写明：从 @image1 自然运动到 @image2，不换人、不换场景、不换衣服。

如果用户强调“基础画面以参考图为准”，必须把 @image1 写成唯一基础画面：严格保持 @image1 的人物、服装、坐姿、场景、光线、构图和背景陈设；不要重新设计背景，不要增加新道具，不要切到剧情画面。只允许自然口播需要的轻微表情、嘴型、手部和身体细微动作。

不要只写“首帧/尾帧”，因为图生视频模型可能无法绑定图片。必须显式写 `@image1`、`@image2`。

## 短时长拆分规则

当用户说 Grok 时长/额度不够、要“拆成两个镜头”、要“分镜头1/分镜头2”、或原镜头信息量超过当前视频时长时，必须把一个原镜头拆成 A/B 两个更单一的子镜头，而不是硬塞进一个 8-10 秒镜头。

命名和输出：

- 用 `镜头{编号}A`、`镜头{编号}B` 命名，例如 `镜头3A｜5秒｜...`、`镜头3B｜5秒｜...`。
- 用户要“第二个镜头分镜头1”时，输出 `镜头2A`；要“第二个分镜头”时，输出 `镜头2B`。
- 每个子镜头只承担一个动作任务：例如 2A 拍“厨房开始忙 + 第一次被打断”，2B 拍“回应一句 + 拿锅铲回头忍住”；3A 拍“一桌小事堆到她手里”，3B 拍“家人站旁边等她处理”。
- 每个子镜头仍然必须使用 `@image1` / `@image2`，并重新定义该子镜头自己的首尾图，不要沿用原完整镜头的尾图导致跨度过大。
- 拆分后，A 的尾帧应能自然接 B 的首帧；B 的尾帧再落到原镜头的情绪终点。
- 不要在 5 秒镜头里强行塞多个人物、多个对白和复杂走位。优先保留旁白、动作和一个情绪变化。

5 秒子镜头推荐结构：

```text
0.0s-1.5s
画面：从 @image1 开始，建立动作和场景。环境声，不要对白。

1.5s-3.2s
画面：核心动作变化。
旁白精确出现：
1.6s-3.1s
旁白说：
“{一句11-16字的故事解释}”

3.2s-5.0s
画面：自然接近 @image2，完成一个明确情绪或动作落点。尽量不要新增对白。
```

如果 5 秒内必须放人物声音：

- 只保留一句短画外音或一句短回应。
- 画外音人物不要入镜，不要对口型。
- 入镜人物开口最多一次；旁白时必须闭嘴。
- 口播超过 16 字时必须压缩，不要靠加快语速解决。

拆分示例：

```text
镜头2A：切菜 → 手机亮 → 第一次被打断。
旁白：“她一进厨房，就像进了全家的后台。”
画外音：“妈，签字！”
林姐不说话，只停一下。

镜头2B：继续做饭 → 低声回应 → 拿锅铲回头忍住。
旁白：“没人问她累不累，只问她东西在哪。”

镜头3A：餐桌琐事摊开 → 桌面更满。
旁白：“一桌小事，都落到她手里。”

镜头3B：家人站旁边等 → 林姐继续处理。
旁白：“不是她记性好，是全家都懒得记。”
```

## 旁白主导规则

岳姐情感故事类视频优先使用“旁白给观众讲故事 + 少量人物关键对白”的结构，而不是短剧式多人连续对白。用户确认过这种形式更适合当前内容。

默认比例：

- 旁白约 70%-85%，负责讲清故事、解释误区、推动情绪。
- 人物对白约 15%-30%，只保留 1 句关键短句，用来制造真实感或情绪转折。
- 每个镜头尽量只有 1 个入镜人物开口；其他人物做倾听、沉默、表情反应或画外音。
- 旁白出现时，所有画面人物嘴巴必须闭着，不要对口型。
- 人物关键对白出现时，必须写清“只有谁嘴唇动，其他人闭嘴听”。
- 如果口播审查发现旁白太长，优先压缩旁白，而不是加快语速。
- 旁白应像在给观众讲故事，不要像广告语、鸡汤、标题党或硬金句。

常用结构：

```text
0.0s-2.3s
画面：从 @image1 开始，人物沉默做动作。

旁白精确出现：
0.2s-2.2s
旁白说：
“{一句18字以内的故事解释}”

说话人控制：
这句是旁白，不是画面人物说。旁白出现时，所有画面人物嘴巴都必须闭着，不要对口型。

2.3s-4.4s
画面：关键人物开口，另一个人物闭嘴听。

口播精确出现：
2.6s-4.2s
{人物名}开口说：
“{一句关键短对白}”

说话人控制：
这句话必须是{人物名}说。画面里只有{人物名}嘴唇轻轻动，其他人物嘴巴闭着，只听，不要对口型。

4.4s-6.2s
画面：沉默反应，动作落下，不要新增对白。

6.2s-8.0s
画面：自然接近 @image2，情绪落点。

旁白精确出现：
6.4s-8.0s
旁白说：
“{一句13字左右的情绪收束}”
```

镜头4确认模板（可作为后续镜头风格基准）：

```text
旁白主导规则：
这个镜头以旁白给观众讲故事为主，人物对白只保留岳姐一句关键短句。
旁白出现时，林姐和岳姐嘴巴都必须闭着，不要对口型。
岳姐说关键对白时，只有岳姐嘴唇轻轻动，林姐必须闭嘴听。
林姐全程不说话，只用抬眼、怔住、手松开体检单、沉默来反应。
不要让两个人同时说话，不要让林姐抢话。
声音比例：旁白约70%，人物对白约30%。

0.2s-2.2s 旁白：
“她一直以为，多扛一点，家里就少乱一点。”

2.6s-4.2s 岳姐开口说：
“那你倒下了，他们就会了吗？”

6.4s-8.0s 旁白：
“她才发现，所有人都在等她。”
```

## 岳姐专门口播总结规则

当用户要“专门给岳姐口播”“岳姐口播控制在30s”“一分钟口播”“10s 10s一个镜头”“口播故事版图片”“根据口播镜头生成故事版”时，优先使用 `references/yuejie-oral-summary-shots.md` 和 `references/yuejie-family-oral-topic-patterns.md`。这些文件记录了本类任务的口播稿、6个10秒镜头拆法、单张岳姐参考图策略、背景规范、60秒字数限制、以及“真口播而不是剧情分镜”的提示词模板。

默认把 30 秒口播拆成 3 个 10 秒镜头，而不是一个长镜头：

```text
镜头10A｜0-10秒｜点破问题
镜头10B｜10-20秒｜算清代价
镜头10C｜20-30秒｜收束观点
```

每段只讲一个信息点，合起来形成“不是她的问题 → 代价是什么 → 家应该怎么变”。总字数控制在 90-105 个中文字符左右，每个 10 秒镜头约 28-36 字。

关键坑：结尾口播必须承接前面故事已经铺过的事实，不要突然引入没有出现过的词或口头禅。例如前文没有“我来吧”，就不要在结尾突然说“你总说我来吧”。优先承接“家里人习惯等她”“她一住院家里才乱”“不是家离不开她，是大家都习惯了等她”。

口播画面控制：画面只有岳姐一个人；只有岳姐嘴唇动；无旁白、无画外音、无其他人物；同一人物、同一服装、同一会客厅；语气平稳克制，不训人、不喊口号。

推荐 10 秒拆分台词：

```text
10A：她不是太能干，是家里人太习惯等她。做饭、找药、管孩子，最后都落在她身上。
10B：她一住院，全家才发现，原来谁都没学会生活。女人最累的不是忙，是所有人都觉得你停不下来。
10C：家不是离不开她，是大家都习惯了等她。家不是靠一个人硬撑，是每个人都要伸手。
```

如果用户要故事版图片，生成 9:16 中文表格图：标题 + 基础信息 + 表格（画面内容、口播台词、镜头运动、情绪重点）+ 底部口播控制。30秒用三行，60秒用六行。若生图文字不清晰，下一版减少表格字数，只保留镜头、主题、口播、画面、情绪，不要继续堆长文本。

重要口播坑位：

- 用户反馈过“感觉不像口播”：口播提示词不要写成复杂导演分镜；要强调固定机位、岳姐面对镜头连续自然说话、不要切镜、不要插入家庭剧情画面。
- 用户反馈过“口播稿莫名其妙”：不要把口播写成剧情复述或硬总结，例如“她一住院，全家才发现，原来谁都没学会生活”容易显假。岳姐口播应借故事讲普遍处境：小事不断找她、长期兜底被看不见、所有人默认她会处理、家事不能只等一个人。
- 前文没有铺过的词不要突然引入。比如没有“我来吧”，结尾就不要硬说“我来吧”。优先承接“家里人习惯等她”“小事都找她”“别只等一个人兜底”。
- 一分钟口播建议6个10秒镜头；全片只要一张岳姐正面半身参考图也可以，前提是固定机位、同人同服同场景。最多加第二张轻微低眼/侧身参考图，不要提供太多角度。

## 说话人控制规则

Grok / 图生视频很容易在多人画面中搞混谁在说话。所有带口播的分镜控制提示词必须显式写说话人控制：

- 如果画面里有两个人，写清每个人在画面哪边、谁开口、谁闭嘴、谁看谁。
- 某一句台词出现时，必须写：`这句话必须是{人物名}说，画面里只有{人物名}嘴唇动。`
- 非说话人必须写：`嘴巴闭着，只听/看/停顿，不要开口。`
- 画外音人物必须写：`只作为画外音，不要出现人物，不要入镜。`
- 旁白必须写：`这句是旁白，不是画面人物说；旁白出现时，画面人物嘴巴必须闭着，不要对口型。`
- 如果用户反馈效果不好，优先减少角色对白；让画面人物只开口一次，其余用画外音、环境声、表情反应和旁白完成。
- 单镜头里不要让多个入镜人物轮流说话。多人在画面时，宁可把另一个人设为倾听者，也不要让模型频繁切换嘴型。

## 基础画面提示词写法

基础画面提示词要包含：

- 9:16 竖屏。
- 真实生活化风格。
- 明亮自然光或柔和室内光。
- 场景和具体生活痕迹。
- 岳姐/中年女人的服装、动作、状态。
- 中老年观众爱看的家庭氛围。
- 禁止项：不要滤镜感、不要夸张演技、不要文字字幕。

示例句式：

```text
9:16竖屏，真实生活化风格，白天自然光，{人物}在{场景}{动作}。
{场景}干净温暖，但有明显生活痕迹：{道具1}，{道具2}，{环境细节}。
{人物}穿{服装}，{关键动作}，动作{熟练/迟疑/疲惫/克制}，明显已经习惯{长期状态}。
整体真实、生活化、中老年观众爱看的家庭氛围，不要滤镜感，不要夸张演技，不要文字字幕。
```

## 镜头运动写法

镜头运动要自然，不炫技。优先使用：

- 从中景开始。
- 缓慢推进到半近景。
- 轻微跟随人物移动。
- 稳定手持感或生活纪录片感。
- 通过推进突出疲惫、停顿、沉默、被消耗。

避免：

- 大幅摇移、快切、航拍、夸张推拉。
- MV 感、广告感、悬疑片感。
- 为了炫技破坏真实家庭氛围。

## 动作设计和秒数口播

必须按镜头时长拆秒数。单镜头不能超过 10 秒。

- 每段 2-3 秒最自然。
- 每段包含一个清楚动作。
- 口播要短，能在对应秒数里自然说完。
- 对白像家里人说话，不要像金句。
- 旁白只在需要点题时出现，不要每段都说教。
- 同一段里不要塞多个角色连续说话。2 秒段落通常只放 1 句短台词，最多再加一个极短回应。
- 如果 4 秒内出现老公、孩子、婆婆、主角连续回应，通常不自然，必须拆段、删台词或改成电话铃/画外声叠加。

9 秒镜头常用拆法：

```text
0s-2s
{第一个动作}

口播：
{角色}：
“{短台词}”

2s-4s
{第二个动作}

口播：
{角色}：
“{短台词}”

4s-6s
{第三个动作}

口播：
{角色}：
“{短台词}”

6s-9s
{最后动作和情绪落点}

旁白：
“{一句真正点题的话}”
```

## 口播时长审查

生成分镜控制提示词后，必须快速审查每段口播是否能在对应秒数里自然说完。不要等用户追问“4秒能说这么多话吗”才检查。

粗略判断规则：

- 普通中文对白自然语速约每秒 4-5 个字；有停顿、情绪、动作干扰时按每秒 3-4 个字估算。
- 2 秒段落适合 6-10 个字左右的短句，例如“袜子呢？”“放桌上。”
- 3 秒段落适合 10-14 个字左右。
- 4 秒段落最多放一到两句短对白，不适合三四个角色轮番说话。
- 主角一边做动作一边说话时，要给动作留空间，台词更要短。
- 旁白如果超过 18-22 个字，通常需要 5 秒以上，或拆到下一个镜头。

必须主动发现并修改这些问题：

- 4 秒里塞进老公、孩子、婆婆三个人问话，再加主角三句回应。
- 同一时间段既有复杂动作，又有多句口播。
- 旁白句子太长，压缩到镜头末尾 2-3 秒里。
- 台词虽然短，但角色切换太多，听感像堆信息。

修改方法：

- 删掉重复信息，只保留最能说明“大家都默认她会处理”的一两句。
- 把多个角色的问话拆到不同时间段。
- 用“手机连续响”“画外传来几句催促声”代替逐句写满。
- 把主角回应压缩成一句，例如“都在老地方。”
- 把长旁白移到 6s-9s，或改短成一句更有力的话。

审查输出可以很短：

```text
口播时长审查：
原版 4 秒内角色和台词过多，不自然。
已压缩为每 2-3 秒一个主要声音点，保留“所有人都默认她会处理”的情绪。
```

## 画面情绪写法

画面情绪要写“这个镜头真正拍什么”，不是简单复述剧情。

常用方向：

- 不是吵闹崩溃，而是所有人都默认她会处理。
- 不是突然委屈，而是长期被消耗后的麻木感。
- 不是控诉家人，而是拍出习惯性付出的重量。
- 不是苦情，而是中年女人把很多事都吞下去的熟练和疲惫。
- 不是夸张表演，而是手没停、嘴在回应、脸慢慢没了表情。
