# 单独图片批量生成

当用户已经看过故事版图片并确认没问题后，通常会要求生成单独素材图。常见口令：

```text
前五个镜头的首尾两帧单独给总共十张图片，每张尺寸9:16，只要内容不要文字。
```

## 执行规则

- Codex image2 一次最多 10 张，所以默认第一批只生成前五个镜头的首帧和尾帧。
- **交付层面建议把 10 张拆成两批生成，每批 5 张**：先生成 1-5，再生成 6-10。这样更稳，避免一次并行太久导致用户新消息打断后只完成部分图片。
- 每张都是独立 9:16 竖屏图，不是故事版拼图，不要分镜表，不要边框。
- 只要画面内容，不要文字、标题、字幕、编号、logo、水印、可读手机屏幕文字。
- 严格使用已确认故事版里的首帧/尾帧提示词，必要时只补充“无文字、单张、9:16、明亮清楚、人物一致”。
- 镜头 6 以后另起下一批，仍然最多 10 张，不要一次请求超过 10 张。
- 生成前先整理本批清单；生成后把图片展示给用户。
- **如果批量生成被用户新消息打断，先明确说明已成功生成几张、哪些编号还没跑；不要假装全部完成。**
- 生成后必须把本次 story 的图片和数据归档到 `storys/` 下的新目录。

## 第一批固定顺序

1. 镜头1首帧
2. 镜头1尾帧
3. 镜头2首帧
4. 镜头2尾帧
5. 镜头3首帧
6. 镜头3尾帧
7. 镜头4首帧
8. 镜头4尾帧
9. 镜头5首帧
10. 镜头5尾帧

## 单图提示词尾部

每张图的提示词尾部统一追加：

```text
单张独立9:16竖屏画面，只要画面内容，不要任何文字、字幕、标题、编号、logo、水印、表格、边框或拼图，画面明亮清楚，适合中老年手机观看。
```

## storys 归档规则

每次 Codex 生成故事版图片或单独图片后，都要在下面目录新建一个 story 子目录：

```text
/home/ubuntu/.hermes/skills/yuejie-story-generate/storys
```

目录命名建议：

```text
YYYYMMDD-HHMM-{短标题拼音或英文slug}
```

如果不知道短标题，用：

```text
YYYYMMDD-HHMM-story
```

每个 story 目录尽量包含：

```text
storyboard.md      # 本次确认的故事版/分镜表
prompts.md         # 每张图对应的 image2 提示词
manifest.json      # 图片清单、镜头编号、首帧/尾帧、生成时间、尺寸、备注
images/            # 生成出来的图片文件
```

图片命名建议：

```text
shot01_first.png
shot01_last.png
shot02_first.png
shot02_last.png
...
```

`manifest.json` 建议结构：

```json
{
  "title": "故事标题",
  "created_at": "YYYY-MM-DD HH:mm",
  "ratio": "9:16",
  "batch": "shots 1-5 first/last",
  "images": [
    {
      "file": "images/shot01_first.png",
      "shot": 1,
      "frame": "首帧",
      "prompt": "对应 image2 提示词",
      "notes": "明亮清楚，无文字"
    }
  ]
}
```

如果当前环境只能展示图片，不能直接拿到图片文件路径，也要先创建 story 目录，并保存 `storyboard.md`、`prompts.md`、`manifest.json`。等图片文件可访问后，再补进 `images/` 并更新清单。
